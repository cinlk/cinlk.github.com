WSGI 协议

7月 19 2015

  wsgi是应用程序或框架与Web服务器之间的一种接口，定义一套接口来实现服务器与应用端的通信规范。当应用程序是按照WSGI规范开发的，它可以应用在任何遵守该规范的服务器上运行。

服务端 应用端 中间件

  应用端：可调用对象，可以是函数，方法，或实现了call的实例；接受两个参数，一个是env参数，一个是start_response；返回一个可遍历的对象。
服务端：服务端为每个http请求，调用应用端的可调用对象，服务端类似容器的概念，为应用端提供运行环境并且负责和web服务器通信。它从web server哪里获取请求，调用应用端，然后在封装应用的返回值，返回response给web服务器。
中间件：同时实现了应用端和服务端的角色，她对于某些应用来说是服务端，而对于某些服务端来时是应用。比如它可以把url请求路由至不同的应用并且重写environ变量。中间件可以不停的嵌套，最终形成一个中间件栈。

paste deploy

  paste deployment 是用于发现核配置WSGI application和server的系统。对于WSGI application 用户提供一个单独的简单的函数(loadapp)，用于从配置文件或者python egg中加载WSGI application。因为WSGI application提供了唯一的单独简单的访问入口，所以application发布者并不需要暴露application的内部的实现细节。
从keystone-paste.ini配置文件来分析配置

[filter:debug]
paste.filter_factory = keystone.common.wsgi:Debug.factory

[filter:build_auth_context]
paste.filter_factory = keystone.middleware:AuthContextMiddleware.factory

[filter:token_auth]
paste.filter_factory = keystone.middleware:TokenAuthMiddleware.factory

[filter:admin_token_auth]
paste.filter_factory = keystone.middleware:AdminTokenAuthMiddleware.factory

[filter:xml_body]
paste.filter_factory = keystone.middleware:XmlBodyMiddleware.factory


[filter:url_normalize]
paste.filter_factory = keystone.middleware:NormalizingFilter.factory

[filter:sizelimit]
paste.filter_factory = keystone.middleware:RequestBodySizeLimiter.factory

[filter:stats_monitoring]
paste.filter_factory = keystone.contrib.stats:StatsMiddleware.factory
#省略部分filter

filter是可调用的对象，接受一个application对象作为参数，返回封装后的application，作为中间件应用

[app:public_service]
paste.app_factory = keystone.service:public_app_factory

[app:service_v3]
paste.app_factory = keystone.service:v3_app_factory

[app:admin_service]
paste.app_factory = keystone.service:admin_app_factory

app应用，文件可以定义多个app，文件中app指需要调用的类
[pipeline:public_api]

# The last item in this pipeline must be public_service or an equivalent
# application. It cannot be a filter.
pipeline = sizelimit url_normalize build_auth_context token_auth admin_token_auth xml_body_v2 json_body ec2_extension revoke_extension user_crud_extension example attribute teacherAttribute public_service

[pipeline:admin_api]
# The last item in this pipeline must be admin_service or an equivalent
# application. It cannot be a filter.
pipeline = sizelimit url_normalize build_auth_context token_auth admin_token_auth xml_body_v2 json_body ec2_extension  s3_extension crud_extension admin_service

pipeline由一系列的filter组成形成中间件链，最后一个是应用，即将前面的filter应用到application。

[composite:main]
use = egg:Paste#urlmap
/v2.0 = public_api
/v3 = api_v3
/ = public_version_api

[composite:admin]
use = egg:Paste#urlmap
/v2.0 = admin_api
/v3 = api_v3
/ = admin_version_api

composite复合应用，由其它多个应用组成。每个key与value对应一个应用。其中use=egg:Paste#urlmap表明使用的是Paste.urlmap。urlmap是一个特殊的composite application，他可以根据请求前缀，调度到其他的application中去。
